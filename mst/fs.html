4、请尽可能多的列举CSS3新增属性和选择器类型？

新增CSS3属性：
  • border-radius 设置所有四个 border-*-radius 属性的简写属性。
  • box-shadow 向方框添加一个或多个阴影。
  • border-image 设置所有 border-image-* 属性的简写属性。
  • background-origin 规定背景图片的定位区域。
  • background-size 规定背景图片的尺寸。
  • background-clip 规定背景的绘制区域。
  • text-shadow 设置文本阴影
  • text-overflow 设置文本溢出时的显示效果
  • @font-face 定义自己的字体
  • transform:
    • translate 水平垂直移动
    • rotate 平面内旋转
    • scale 缩放
    • skew 斜切
  • translate3d(x,y,z) 定义 3D 转化。
  • scale3d(x,y,z) 定义 3D 缩放转换。
  • rotate3d(x,y,z,angle) 定义 3D 旋转。
  • perspective(n) 定义 3D 转换元素的透视视图。
  • transition 设置CSS动画过渡
  • @keyframes 动画
  • box-sizing 设置盒模型的尺寸计算行为，
    • content-box : 默认和盒模型尺寸，盒子实际尺寸 = width(或者height) + padding + border-width
    • border-box : CSS3盒模型尺寸，盒子实际尺寸 = width(或者height)
    • outline-offset 对轮廓进行偏移，并在超出边框边缘的位置绘制轮廓。
  • flexible box 弹性盒子模型
新增CSS3选择器：
  • E[foo^=”bar”]：匹配foo属性值以”bar”开始的E元素
  • E[foo$=”bar”]：匹配foo属性值以”bar”结束的E元素
  • E[foo*=”bar”]：匹配foo属性值包含”bar”的E元素
  • E:root：匹配E所在文档的根元素
  • E:nth-child(n)：匹配元素类型为E且是父元素的第n个子元素
  • E:nth-last-child(n)：匹配元素类型为E且是父元素的倒数第n个子元素（与上一项顺序相反）
  • E:nth-of-type(n)：匹配父元素的第n个类型为E的子元素
  • E:nth-last-of-type(n)：匹配父元素的倒数第n个类型为E的子元素（与上一项顺序相反）
  • E:first-of-type：匹配父元素的第一个类型为E的子元素
  • E:last-child：匹配元素类型为E且是父元素的最后一个子元素
  • E:last-of-type：匹配父元素的最后一个类型为E的子元素
  • E:only-child：匹配元素类型为E且是父元素中唯一的子元素
  • E:only-of-type：匹配父元素中唯一子元素是E的子元素
  • E:empty：匹配不包含子节点(包括文本)的E元素

5、用代码实现数组去重？

a、通用方法,哈希表方法
function make_unique1(arr)
{
   var rsarr=[];
   var obj = {};
   for(var i=0;i<arr.length;i++)
   {
      if(!obj[arr[i]])
      {
        obj[arr[i]] = true;
        rsarr.push(arr[i]);
      }
   }
   return rsarr;
}
b、indexOf方法,查找新建的数组里面有没有放入相同的元素
function make_unique2(arr)
{
   var rsarr=[];

   for(var i=0;i<arr.length;i++)
   {
      if(rsarr.indexOf(arr[i])==-1)
      {       
        rsarr.push(arr[i]);
      }
   }
   return rsarr;
}
c、indexOf方法,查找元素的index值是不是出现在第一次出现的地方，只将第一次出现的元素放入新数组
function make_unique3(arr)
{
   var rsarr=[];

   for(var i=0;i<arr.length;i++)
   {
      if(arr.indexOf(arr[i])==i)
      {       
        rsarr.push(arr[i]);
      }
   }
   return rsarr;
}
d、从数组的第二个元素依次往前比较，如果出现与前面相同的，就将当前的元素删掉
function make_unique4(arr)
{
   for(var i=1;i<arr.length;i++)
   {
      for(j=i-1;j>=0;j--)
      {
        if(arr[i]==arr[j])
        {
          arr.splice(i,1)
          i--;
        }
      }
   }
   return arr;
}

6、用代码实现getByClass？

function getByClass(oParent,sClass)
{
     if(oParent.getElementsByClassName)
     {
        return oParent.getElementsByClassName(sClass);
     }

    var aEle = oParent.getElementsByTagName('*');
    var re = new RegExp('\\b'+sClass+'\\b');
    var rs = [];

     for(var i=0;i<aEle.length;i++)
     {
        if(re.test(aEle[i].className))
        {
           rs.push(aEle[i]);
        }
     }

    return rs;
}

9、什么是事件委托？

事件委托：利用事件冒泡机制，在顶层的元素上绑定一个处理函数。当在子元素上发生点击、悬停等事件时，该事件通过事件冒泡传递到顶层，顶层元素的处理函数通过判断发生事件的子元素的特征，给予对应的处理方法。实现代码如下：

function f2(d) 
{
    document.onclick = function(e) 
    {
          var e = e || window.event;
          var target = e.srcElement || e.target;
          if (target.tagName == “LI”)
          {
              ……
          }
    }
 }

e.srcElement || e.target: 获取事件发生时，最初的那个dom元素。
通过对元素属性的判断，例如标签名，ID ,CLASS等等，来执行不同的处理函数。
这样做的好处是把绑定在多个子元素上的事件统一委托给顶级元素，可以很大的提高性能，另外，新增的子元素会根据特征自动地关联对应的处理方法。

10、什么是闭包？

定义在一个函数里面的函数，外部函数可以叫父函数，内部的函数可以叫子函数，子函数可以访问父函数的局部变量，这个子函数就是一个闭包。

闭包的作用：
a、创建匿名自执行函数，函数内变量和方法不会污染全局,比如：

(function(){    
    var els = document.getElementsByTagName('li');
    for(var i = 0,lng = els.length;i < lng;i++){
        els[i].style.color = 'red';
    }    
})();

b、创建私有化变量，实现封装/模块化代码

var person= function(){    
    //变量作用域为函数内部，外部无法访问    
    var name = "default";       

    return {    
       getName : function(){    
           return name;    
       },    
       setName : function(newName){    
           name = newName;    
       }    
    }    
}();
console.log(person.name);//直接访问，结果为undefined    
console.log(person.getName());  //default 
person.setName("jozo");    
console.log(person.getName());  //jozo

闭包的缺点：
闭包会占用更多的内存(闭包可以让变量常驻内存)
闭包会引起内存泄露(循环引用，如下：)

window.onload = function(){
    var el = document.getElementById("id");
    el.onclick = function(){
        alert(el.id);
    }
}

解决方法：

window.onload = function(){
    var el = document.getElementById("id");
    var id = el.id; //解除循环引用
    el.onclick = function(){
        alert(id); 
    }
    el = null; // 将闭包引用的外部函数中活动对象清除
}

12、前端性能优化总结？

一、部署：

1、前端代码的压缩与合并，使用GZIP压缩。
2、代码资源放置于不同域名下的服务器，图片及样式等静态资源使用无cookie域名。
3、通用静态资源服务器内存缓存。
4、使用内容分发网络 CDN。
5、为文件头指定Expires设置浏览器缓存时间。
6、favicon.ico要小而且可缓存。
7、权衡DNS查找次数
8、避免不必要的重定向
9、配置ETags
